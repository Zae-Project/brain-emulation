Documentation 11 · Troubleshooting & Verification

Even with a polished UI, complex simulations occasionally hit snags. This chapter lists common issues, diagnostic techniques, and verification steps to ensure your changes preserve expected behavior.

Quick Diagnostics

  • Console Errors — Press F12 or open developer tools. Most configuration issues log descriptive errors (e.g., missing template fields).
  • Network Tab — Confirm manifest.json and template files load with HTTP 200.
  • Banner Notifications — Import/export actions display status banners at the bottom of the screen.

Common Issues

No Neurons Rendered

  • Check that networkSize is > 0 (procedural mode).
  • Ensure the selected template has at least one cluster and neuron group.
  • Look for JSON parse errors when loading templates.

Inspector Shows “—” for Values

  • Verify neuron presets include biophysical parameters (threshold, leak).
  • Ensure neuron types reference known slugs.

Import Fails

  • Run the template through SNN_CONFIG_IO.deserializeTemplate in the console to view validation errors.
  • Check for trailing commas or comments; JSON must be strict.

Verification Checklist

Before committing changes, step through:

  • Select each preset and ensure the canvas populates without errors.
  • Click multiple neurons; confirm taxonomy labels and glyphs align.
  • Export and re-import a template to test round-tripping.
  • Toggle all sliders and buttons to confirm UI bindings.
  • Run the manifest refresh script and reload to verify auto-discovery.
Automated Checks (Roadmap)

Future scripts will validate template schema (npm run test:templates) and run visual regression tests for the inspector. Track progress in docs/dev_progress_tracker.md.

Reporting Bugs

  • Include template files if the issue arises after import.
  • Provide console logs and browser version.
  • Mention whether you modified manifest.json or core JS files.
